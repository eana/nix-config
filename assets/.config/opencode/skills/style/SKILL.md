description: >
  Personal style rules for all written output: code, comments, documentation, and commit messages
compatibility: opencode
---

## Emojis

- Never use emojis anywhere: code, comments, documentation, commit messages, or
  any other written output

## Commit messages

- Do not use emoji prefixes (no gitmoji style)

## Verb tense

- Use present tense for general descriptions and instructions
- Use imperative mood for commands and step-by-step instructions
- Use past tense only for completed actions or historical context
- Use future tense for expected outcomes or planned actions

## Mood

- Prefer indicative mood for factual statements
- Use imperative mood for instructions and commands
- Use subjunctive mood only for recommendations or hypothetical scenarios

## Pronouns

- Use second person ("you") for instructions and guidance
- Avoid first person ("I", "we") unless providing author notes or explanations
- Use third person for describing system behaviors or features

## Technical terms

- Use precise technical terms, tool names, and domain-specific vocabulary
  consistently
- Introduce new or uncommon terms with a brief explanation if needed

## Structure and style

- Organise content into clear sections with headings
- Use step-by-step guides, prerequisites, and configuration details where
  appropriate
- Include code blocks, configuration snippets, and command examples for clarity
- Reference external resources or documentation as needed
- Emphasise reproducibility, automation, and security best practices
- Maintain consistent terminology and formatting throughout

## Visual aids

- Reference or include diagrams and screenshots to support explanations when
  helpful
